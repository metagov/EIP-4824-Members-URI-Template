API Documentation

This page provides detailed documentation for the API endpoints available in the application, focusing on retrieving unique voters for a specific DAO space.

Endpoints

  • GET /members/<space> - Fetches unique voters for a specified DAO space.

    Parameters:

    • space: String - The DAO space identifier (e.g., "beets.eth") as part of the URL path.
    • cursor: Integer (optional) - Used to paginate results based on this cursor. If provided, the API fetches results created after the timestamp denoted by this cursor.

    Description:

    This endpoint retrieves a list of unique voters from the specified DAO space, allowing for pagination if a cursor is provided. It is effective for accessing data in manageable segments when dealing with large datasets.

    Example Usage

    Request without cursor:

    GET /members/beets.eth

    Response:

    {
  "members": [
    { "id": "0x123...", "type": "EthereumAddress" }
  ],
  "next_cursor": null,
  "@context": "http://daostar.org/schemas",
  "type": "DAO",
  "name": "beets.eth"
}

    Request with cursor:

    GET /members/beets.eth?cursor=1609459200

    Response:

    {
  "members": [
    { "id": "0x456...", "type": "EthereumAddress" }
  ],
  "next_cursor": 1609462800,
  "@context": "http://daostar.org/schemas",
  "type": "DAO",
  "name": "beets.eth"
}
